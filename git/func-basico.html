Git

Funcionamento básico

O Git permite o controle de versões de repositórios.

Um repositório é o diretório onde estão os arquivos e pastas de um projeto.

Para gravar uma versão do repositório, deve ser executado um commit.

Para executar commits, o git deve ser iniciado no repositório. Para isso, no diretório raiz do projeto, em um sistema com o Git instalado, execute: git init. Neste ponto foi criada uma pasta oculta com o nome .git que armazena os arquivos do versionamento.

Arquivos e pastas em um repositório podem estar:

  1. Inalterados desde o último commit.
  2. Alterados desde o último commit e prontos para serem commitados (stageds).
  3. Alterados desde o último commit (untrackteds).

Como ainda não foi realizado nenhum commit, nenhuma versão do projeto ainda foi salva. Isso pode ser verificado executando: git status

Todo o conteúdo do repositório é apresentado em vermelho e descrito como untrackted. Para preparar todo o conteúdo para um commit, execute: git add *

Agora o conteúdo é apresentado em verde e descrito como alterações a serem commitadas. Para realizar o primeiro commit, execute: git commit -m "Mensagem do primeiro commit". A mensagem é obrigatória e deve descrever a principal alteração no projeto do commit.

Para ver todos os commits que já foram realizados em um repositório, execute: git log
