Git

Repositórios remotos

O Git permite que um repositório local seja copiado em máquinas remotas, que atuam como servidores. Uma das formas de trabalhar com repositórios remotos é:

  1. Criar um repositório na máquina remota com git init
  2. Copiar o repositório remoto na máquina local com git clone
  3. Realizar os commits no repositório local e atualizar o repositório remoto com git push
  4. Atualizar o repositório local com alterações no repositório remoto usando git pull

Github

Github é uma plataforma para compartilhamento de repositórios Git.

Para realizar alterações em um repositório compartilhado no Github, um desenvolvedor dever realizar um fork desse repositório, o que vai criar uma cópia do repositório em sua própria conta na plataforma.

Para atualizar o repositório original com as suas alterações, o desenvolvedor deve solicitar um pull request.

Para atualizar a sua cópia com possíveis alterações no repositório original, é preciso definir o repositório original como um remoto do seu repositório local. Para isto, execute:

git remote add upstream endereco_repositorio_original

Quando quiser atualizar, execute git fetch upstream, para baixar as atualizações para seu repositório local, e git merge upstream/master, para mesclar as alterações no seu repositório local. Para atualizar a sua cópia no Github, execute git push.
